# MEMORY_RULES.md - 记忆维护规则

这份规则用于指导大钧哥如何维护工作区内的短期与长期记忆。

## 1. 每日记录写到哪里

日常发生的事情、对话里的新信息、临时决定、待办线索，优先写入：

- `memory/YYYY-MM-DD.md`

适合写入每日记录的内容：

- 用户当天说过的重要信息
- 新的偏好或习惯
- 临时计划、想法、待办
- 值得后续跟进的上下文
- 当天完成过的重要操作

原则：先记下来，宁可稍微多一点，也不要漏掉关键上下文。

## 2. 长期记忆写到哪里

稳定、不容易变化、未来大概率还会反复用到的信息，写入：

- `MEMORY.md`

适合写入长期记忆的内容：

- 用户喜欢的称呼方式
- 用户长期偏好
- 助手身份设定
- 反复出现的重要项目背景
- 对未来判断很有帮助的稳定事实

原则：长期记忆要精炼，避免把流水账直接塞进去。

## 3. 什么时候更新每日记录

遇到以下情况时，优先更新当天的 `memory/YYYY-MM-DD.md`：

- 用户明确说“记住这个”
- 建立了新的偏好、规则、习惯
- 完成了重要配置或环境变更
- 做出了后续会影响决策的选择
- 出现了需要后续跟进的事项

## 4. 什么时候整理进长期记忆

当某条信息满足下面任一条件时，可以从每日记录提炼进 `MEMORY.md`：

- 三天后看仍然重要
- 后续大概率会再次用到
- 能帮助更好地理解用户或工作方式
- 是已经稳定下来的偏好或设定

## 5. 不要记什么

以下内容不要轻易写入记忆文件：

- 明文密码、token、密钥
- 不必要的隐私细节
- 一次性、很快过期的信息
- 没确认真实性的猜测

如果必须提到敏感事项，也只记录结论，不记录凭证原文。

## 6. 维护节奏

建议采用以下节奏：

- 日常：随任务顺手更新当天 `memory/YYYY-MM-DD.md`
- 定期：隔几天回看 daily memory，提炼更新 `MEMORY.md`
- 大改动后：做一次 commit，并在可用时 push 到 GitHub

## 7. 写作风格

- 简洁
- 具体
- 可检索
- 少空话
- 以未来复用为目标

写记忆不是写作文，是给未来的自己留线索。
